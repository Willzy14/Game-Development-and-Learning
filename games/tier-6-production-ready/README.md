# Tier 6: Production-Ready

## Learning Objectives
Build a complete, original game from concept to release:
- Original game design and iteration
- Full production pipeline
- Quality assurance and bug tracking
- Player feedback integration
- Build pipelines and deployment
- Marketing and presentation
- Post-launch support planning

## Target Games (1-2 major projects)

### The Goal
Create an **original game concept** that showcases all skills learned:
- Unique mechanics or twist on existing genres
- Professional level of polish
- Complete game from start to finish
- Ready for actual release (itch.io, Steam, mobile stores)

### Project Scope
- 3-6 months of development
- Clear beginning, middle, and end
- Marketing materials (trailer, screenshots, description)
- Playtesting with external players
- Iteration based on feedback

## Technology Recommendation
- Unity with full release pipeline
- Version control best practices (Git Flow or trunk-based)
- Automated testing (if applicable)
- Analytics integration (optional)
- Crash reporting (if applicable)

## Success Criteria
- [ ] Game design document completed and refined
- [ ] All core systems implemented and tested
- [ ] External playtest sessions conducted
- [ ] Game is balanced and tuned
- [ ] Full audio and visual polish
- [ ] Builds successfully for target platform(s)
- [ ] Marketing materials created
- [ ] Game is actually published/released

## Common Pitfalls at This Tier
- Analysis paralysis on game concept
- Scope too large for realistic completion
- Not getting feedback early enough
- Perfectionism preventing launch
- Ignoring marketing until it's too late

## Skills That Should "Click"
- How to validate game ideas quickly
- Importance of vertical slice development
- How to cut scope without losing vision
- Production management and scheduling
- When to call a game "done"

## Production Skills to Master
- Project management (tracking tasks, milestones)
- Bug triage and prioritization
- Build automation
- Release planning
- Community management basics
- Marketing and presentation

## The Portfolio Goal
By the end of this tier, you should have:
1. A shipped game you're proud of
2. A comprehensive portfolio of projects showing progression
3. Deep technical knowledge of Unity and C#
4. The ability to take any game concept from idea to release
5. A documented learning journey that showcases growth

## Success Metrics
- Game is actually released/published
- Positive player feedback received
- Technical challenges were overcome
- All systems from previous tiers integrated successfully
- You can articulate what you learned across the entire journey
