### Problem 1: Web Audio API Autoplay Policy

### Problem 2: Sound Click/Pop Artifacts
- **Context**: Initial sounds had clicking/popping at start and end
- **Error/Issue**: Abrupt volume changes cause audio artifacts
- **Solution**: Used gain envelopes with linearRampToValueAtTime for fade in/out
- **Why It Worked**: Gradual volume changes eliminate clicks
- **Source**: Web Audio API best practices
- **Time Spent**: 20 minutes

### Problem 3: Understanding Oscillator Types
- **Context**: Wanted different sound characteristics
- **Error/Issue**: All sounds sounded the same
- **Solution**: Learned about sine, square, sawtooth, triangle waves
- **Why It Worked**: Different waveforms create different timbres
- **Source**: Web Audio API documentation
- **Time Spent**: 10 minutes

### Problem 4: Ball Getting Stuck in Paddle (Breakout)
- **Context**: Ball collision with paddle triggering multiple times
- **Error/Issue**: Ball could get trapped inside paddle
- **Solution**: Added `dy > 0` check to only detect when ball moving down
- **Why It Worked**: Prevents collision detection on ball's upward journey
- **Source**: Own debugging and logic
- **Time Spent**: 10 minutes

### Problem 6: Enemy Formation Movement (Space Invaders)
- **Context**: Needed enemies to move as coordinated group
- **Error/Issue**: Individual enemy edge detection caused desync
- **Solution**: Flag-based approach - check all enemies, then move group together
- **Why It Worked**: Separates detection from action, maintains formation
- **Source**: Classic Space Invaders behavior study
- **Time Spent**: 20 minutes

### Problem 7: Bullet Spam Prevention
- **Context**: Players could hold spacebar and create bullet stream
- **Error/Issue**: Made game too easy, performance concerns
- **Solution**: Cooldown timer (250ms between shots)
- **Why It Worked**: Forces timing strategy, prevents performance issues
- **Source**: Common game design pattern
- **Time Spent**: 5 minutes

### Problem 8: When to Extract to Shared Library?
- **Context**: Audio code copied three times
- **Error/Issue**: Violating DRY principle, maintenance nightmare
- **Solution**: Applied Rule of Three - extract after third use
- **Why It Worked**: Had enough experience to know what varies vs what's stable
- **Source**: Refactoring best practices
- **Time Spent**: 45 minutes (extraction + documentation)
